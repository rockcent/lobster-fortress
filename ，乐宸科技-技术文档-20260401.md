## 五、AI 服务架构

### 5.1 AI 模型

| 模型 | 用途 | 供应商 |
|------|------|--------|
| `gemini-3.1-pro-preview` | BD 谈判模拟、长文本生成（MoU/PitchDeck） | 火山方舟（Google Gemini） |
| `gemini-3-flash-preview` | 快速提取、结构化需求分析 | 火山方舟（Google Gemini） |

### 5.2 赞弹秀数据流（已修复）

```
用户点击"热点" → fetchBraveTrendingNews() 
  → Brave Search API 获取真实新闻标题 
  → Gemini 生成争议话题 
  → 用户看到真实热点（不再是无数据的 mock fallback）
```

> ⚠️ 如果热点抓取失败，用户会看到明确错误提示而非假数据。

### 5.3 AI 营销云核心流程

```
extractMarketingNeeds()     # 自然语言 → 结构化营销需求
  ↓
generateMatchmaking()       # 需求库 → 跨界资源配对方案
  ↓
simulateBDNegotiation()     # AI 模拟商务谈判（80字以内，不超过预算）
  ↓
refineMatchmakingPlan()     # 细化执行方案（Markdown 格式）
generatePitchDeck()         # 生成合作提案
generateMoU()              # 生成 MOU 协议
generateTimeline()          # 生成执行时间线
```

### 7.1 新增页面到 luoxiang
1. 在 `src/components/` 创建 `NewPage.tsx`
2. 在 `src/App.tsx` 添加 import 和 `<Route>`
3. 构建后 rsync 部署

3. 在 `src/App.tsx` 调用
4. 构建 + rsync 部署到 ai-marketing
